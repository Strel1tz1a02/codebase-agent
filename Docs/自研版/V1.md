## V1：项目结构扫描器

### 1. 当前模块目标

先不做复杂 Agent，只做一个可运行的本地仓库结构扫描器。

这个模块要解决的问题：
- 给定一个本地项目路径，快速得到可读的项目结构信息。
- 产出稳定、可验证的结构化结果，为后续 RAG、Tool Calling、LangGraph 编排提供输入。

它在 `codebase-agent` 项目中的位置：
- 属于基础能力层（文件系统扫描与结构化摘要）。
- 是后续所有“代码理解”能力的上游数据来源。

---

### 2. 范围定义

`In Scope`（V1 要做）：
- 扫描本地目录并构建项目目录树。
- 统计文件总数。
- 统计文件类型分布（按扩展名）。
- 识别主要目录（如 `src`、`tests`、`docs`、`config`）。
- 基于规则识别可能的入口文件。

`Out of Scope`（V1 不做）：
- RAG（切分、向量化、检索）。
- 自然语言问答。
- LLM 参与分析。
- 多轮 Agent 状态管理与图编排。
- 远程仓库下载/克隆。

---

### 3. 输入与输出契约

输入：
- `repo_path: str`（本地项目绝对路径或相对路径）

输出（结构化）：
- `repo_path: str`
- `tree: str`（文本目录树）
- `file_count: int`
- `file_types: dict[str, int]`（例如 `.py: 10`）
- `key_dirs: list[str]`
- `entry_candidates: list[str]`
- `ignored_paths: list[str]`

输出（展示层）：
- 由结构化结果渲染出的 Markdown 报告。

---

### 4. 忽略规则

默认忽略目录：
- `.git`
- `node_modules`
- `__pycache__`
- `.venv`
- `dist`
- `build`

默认忽略文件：
- `.DS_Store`

说明：
- V1 先不解析 `.gitignore` 语义，只使用内置忽略规则。
- 后续版本再支持 `.gitignore`。

---

### 5. 入口文件识别规则（V1）

命中以下文件名即加入候选：
- `main.py`
- `app.py`
- `__main__.py`
- `manage.py`
- `run.py`

路径优先级建议：
- `src/` 下命中优先于根目录命中。

---

### 6. 错误处理与边界

错误处理：
- 路径不存在：返回明确错误信息。
- 路径不可读：返回权限错误信息。
- 路径不是目录：返回类型错误信息。

边界约束（建议）：
- 最大扫描文件数：`20,000`
- 最大目录深度：`20`
- 超限后停止深入，并在结果中提示 `truncated: true`

---

### 7. 推荐文件位置

- `src/tools/file_tools.py`：扫描、树构建、统计、入口识别
- `src/utils/ignore.py`：忽略规则
- `src/main.py`：CLI 入口与结果输出

---

### 8. 函数级任务拆分（先设计，后实现）

建议函数：

1. `scan_files(repo_path: str) -> tuple[list[str], list[str]]`
- 输入：项目路径
- 输出：被纳入扫描的文件列表、被忽略路径列表
- 作用：统一收集文件，应用忽略规则

2. `build_file_tree(files: list[str], repo_path: str) -> str`
- 输入：文件列表、项目根路径
- 输出：目录树文本
- 作用：生成可读项目结构

3. `count_file_types(files: list[str]) -> dict[str, int]`
- 输入：文件列表
- 输出：扩展名统计字典
- 作用：识别项目技术栈分布

4. `find_entry_candidates(files: list[str]) -> list[str]`
- 输入：文件列表
- 输出：入口候选路径列表
- 作用：为后续“项目入口理解”提供线索

5. `summarize_key_dirs(files: list[str]) -> list[str]`
- 输入：文件列表
- 输出：主要目录列表
- 作用：快速定位项目核心区域

6. `generate_v1_report(...) -> str`
- 输入：结构化分析结果
- 输出：Markdown 报告
- 作用：将机器结果转为人可读结果

---

### 9. 验收标准（DoD）

使用测试仓库：
- `E:\projects\test_project_v1`

通过标准：
- 能输出目录树。
- 忽略目录（`.git`、`node_modules`、`__pycache__`、`.venv`）不进入统计。
- 能输出文件总数和文件类型统计。
- 能识别入口候选（至少识别 `src/main.py`、`src/__main__.py`）。
- 能输出主要目录列表（至少包含 `src`、`tests`、`docs`）。

---

### 10. 在 Agent 系统中的角色

V1 模块归类：
- 文件工具
- 工程辅助模块

它的定位：
- 为后续检索模块、RAG 组件、LangGraph 节点提供稳定输入数据。

---

### 11. 对应经典设计思想（简版映射）

- LangChain：`tools` 中的文件系统工具思想（先拿可控上下文，再做推理）。
- LangGraph：将“扫描结果”作为 `state` 的上游输入，后续可接节点流转。
- LlamaIndex：先做文档/代码收集与结构化，再进入索引和检索阶段。
- OpenAI Agents SDK：工具调用前置的数据准备与可追踪中间结果。

---

### 12. 后续升级路径

V1 -> V2：
- 在 V1 文件收集基础上加入代码切分、向量索引、检索问答（RAG）。

V2 -> V3：
- 引入 Tool Calling，让模型按问题自动选择 `list/read/search/retrieve`。

V3 -> V4：
- 使用 LangGraph 管理 `analyze -> decide -> execute -> answer` 流程与循环控制。
