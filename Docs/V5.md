# V5: RAG 工具集成

## 当前模块目标

V5 的目标是把 V2 已经实现的 RAG 检索能力接入 Agent 工具层，让 `--ask-mode agent` 和 `--ask-mode graph` 都可以通过工具调用使用语义代码检索。

本模块位于 Agent 系统的 Tool Calling 层和 RAG 组件之间：RAG 层负责检索代码 chunk，工具层负责把检索能力暴露给 Agent 决策器。

## 推荐文件位置

本阶段涉及文件关系如下：

1. `src/rag/retrieval.py`
   - 已有 RAG 检索入口 `retrieve_relevant_chunks()`。
2. `src/agent/tools.py`
   - 新增 `retrieve_code()`，把 RAG 检索包装成 Agent 工具。
3. `src/agent/adapter.py`
   - 更新决策 prompt，让 LLM 知道何时选择 `retrieve_code`。
4. `src/main.py`
   - 更新 History 打印，让 RAG 命中项显示路径、行号范围、score 和内容预览。
5. `tests/`
   - 覆盖工具注册、工具执行、Agent history 流转和 CLI 展示。

## 代码实现

V5 新增工具：

```python
def retrieve_code(arguments: dict[str, object]) -> dict[str, object]:
    ...
```

工具参数：

```json
{
  "repo_path": "E:\\projects\\codebase-agent",
  "query": "agent loop tool execution",
  "top_k": 5,
  "reindex": false
}
```

工具返回：

```json
{
  "query": "agent loop tool execution",
  "top_k": 5,
  "matches": [
    {
      "relative_path": "src/agent/controller.py",
      "start_line": 9,
      "end_line": 70,
      "content": "...",
      "score": 0.91
    }
  ]
}
```

## 函数解释

### `retrieve_code(arguments)`

输入：

1. `repo_path`：要分析的仓库路径。
2. `query`：用于语义检索的查询文本。
3. `top_k`：返回 chunk 数量，默认 5。
4. `reindex`：是否强制重建索引，默认 false。

输出：

1. `query`：实际检索语句。
2. `top_k`：实际返回上限。
3. `matches`：命中的代码 chunk，包含路径、行号范围、内容和分数。

作用：

把已有 RAG 检索能力包装成 Agent 可调用工具。

为什么需要：

V3/V4 Agent 原来只能使用 `repo_summary`、`read_file` 和 `search_code`。这些工具适合明确关键词或明确文件路径的问题，但不适合“这个流程是怎么工作的”这类语义问题。`retrieve_code` 让 Agent 可以先召回相关代码片段，再基于引用回答。

## 在 Agent 系统中的角色

`retrieve_code` 属于：

1. Tool Calling 工具。
2. RAG 组件适配层。
3. 基于引用的幻觉控制基础能力。

## 对应经典项目设计思想

1. LangChain
   - 类似 Retriever Tool：把检索器包装成工具，交给 Agent 决策是否调用。
2. LangGraph
   - 类似 graph 节点中的 tool execution：模型先决策，图节点再执行工具，并把结果写回 state/history。
3. LlamaIndex
   - 类似 QueryEngine/Retriever 的结果节点：返回带 metadata 的文本片段，供答案生成引用。
4. OpenAI Agents SDK
   - 类似 function tool：工具有明确名称、参数和结构化返回，Agent 通过工具结果继续推理。

## 后续如何升级

1. V6 可以把 `retrieve_code` 的结果写入结构化 trace。
2. V7 可以通过 API 暴露仓库索引和会话问答。
3. V8 可以把 `retrieve_code` 纳入显式 `ToolSpec`，增加 input schema 和 permission。
4. V9 可以基于 `matches` 计算 retrieval hit rate 和 citation coverage。
